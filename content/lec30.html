Lecture 30: JSON vs Object & String Methods

Summary

المحاضرة دي كانت عن الفرق بين JSON وObject في الجافاسكريبت، وإزاي نستخدم كل واحد فيهم حسب الموقف. كمان اتكلمنا عن مجموعة من دوال النصوص (string methods) اللي بتساعدنا نحلل أو نعدّل النصوص قبل حفظها أو إرسالها. الهدف إنك تفهم إمتى تستخدم الـ Object للتعامل المنطقي، وإمتى تستخدم الـ JSON للتخزين أو نقل البيانات.

Key Points

  1. ما هو JSON?

    JSON اختصار لـ JavaScript Object Notation، وده عبارة عن text format بسيط لتخزين أو إرسال البيانات. بيتشابه جدًا مع الـ object لكن بيكون مكتوب كنص مش ككود.

    بيستخدم لما تحب:

    • تحفظ البيانات في ملف.
    • ترسل أو تستقبل بيانات من API أو موقع تاني.

  2. ما هو Object?

    الـ Object هو هيكل بيانات حقيقي بتتعامل معاه جوه كود الجافاسكريبت مباشرة. تقدر تعمل عليه عمليات حسابية، منطقية، وتتعامل مع خصائصه بحرية.

    مثال:

    var obj = { name: "islam", age: 29 };
    • تقدر توصله كده: object.name islam.

  3. الفرق بين JSON و Object:

    الجدول التالي بيوضح الفرق الأساسي بينهم:

    • JSON عبارة عن نص (string) مش كود.
    • Object بيانات حقيقية جوه الكود.
    • الـ JSON ما ينفعش يحتوي على function أو undefined.
    • الـ Object ينفع يحتوي على أي نوع من الداتا.

    بمعنى بسيط: Object بنستخدمه علشان “نعالج البيانات”، لكن JSON بنستخدمه علشان “نخزن أو ننقل البيانات”.

  4. الفرق في طريقة الكتابة (Syntax):

    رغم إن الشكل العام شبه بعض، في فرق بسيط بين كتابة object وJSON:

    • Object Syntax:

      var user = { name: "islam", age: 25 };
      المفاتيح (keys) مش لازم تتحط بين علامات تنصيص.

    • JSON Syntax:

      {
  "name": "islam",
  "age": 25
}
      كل المفاتيح والقيم النصية لازم تتحط بين double quotes.

    ⚠️ لو نسيت تحط علامات تنصيص حوالين المفاتيح في JSON، الكود مش هيشتغل لإنه بيتعامل كنص مش كـ JavaScript object.

  5. التحويل بين JSON و Object:

    عندنا دالتين أساسيتين للتحويل بينهم:

    • JSON.stringify(object) بتحول الـ object إلى نص JSON.
    • JSON.parse(jsonString) بتحول نص JSON إلى object حقيقي.

    مثال:

    var person = { name: "islam", age: 29 };
var jsonData = JSON.stringify(person);
var backToObj = JSON.parse(jsonData);

  6. أنواع البيانات اللي JSON بيدعمها:

    • Number 40
    • String "text"
    • Boolean true / false
    • null
    • Array [1, 2, 3]
    • Object {"name": "islam"}

    أما القيم الغير مدعومة زي undefined أو function فمش هتشتغل في JSON.

  7. String Methods:

    دي مجموعة دوال بنستخدمها علشان نتعامل مع النصوص في JavaScript، وكل دالة منهم بترجع قيمة جديدة من غير ما تغيّر النص الأصلي.

    • split(separator) بتقسم النص إلى Array حسب الفاصل اللي انت بتحدده.
    • slice(start, end) بتقص جزء معين من النص حسب البداية والنهاية اللي تحددها.
    • trim() بتحذف المسافات الزيادة من أول وآخر النص.
    • indexOf(value) بتدور على كلمة أو حرف في النص وتجيب مكانه (أول ظهور ليه).
    • toLowerCase() بتحوّل كل الحروف لـ small letters.
    • toUpperCase() بتحوّل كل الحروف لـ CAPITAL letters.

    أمثلة:

    • var str = " Hello World "
    • str.trim() "Hello World"
    • str.slice(0, 5) "Hello"
    • str.split(" ") ["", "Hello", "World", ""]
    • str.indexOf("World") 6
    • str.toUpperCase() " HELLO WORLD "
    • str.toLowerCase() " hello world "

    💡 كل دالة من دول مش بتغيّر النص الأصلي، لكن بترجع نسخة جديدة بعد التعديل.

Notes

  • 🧩 الفرق الأساسي إن JSON ممكن يتعرض في المتصفح مباشرة كبيانات نصية، لكن Object لا يمكن عرضه إلا بعد تحويله باستخدام JSON.stringify().
  • 💡 استخدم JSON.stringify() لما تحب تحفظ أو ترسل بيانات.
  • ⚙️ استخدم JSON.parse() لما تستقبل بيانات JSON من مصدر خارجي.
  • 📦 دايمًا افتكر إن JSON نص، لكن object داتا حقيقية في الكود.
  • 🧠 الفكرة إن object بيخدم المنطق، والـ JSON بيخدم التخزين.
