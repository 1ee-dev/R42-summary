Lecture 24: Media Queries & CSS Selectors / Pseudo Elements

Summary

المحاضرة دي كانت مليانة معلومات أساسية عن تصميم المواقع المتجاوبة (Responsive Design) باستخدام Media Queries، واتكلمنا كمان عن كل أنواع الـ Selectors في CSS والـ Pseudo Classes وPseudo Elements. الهدف إننا نقدر نتحكم في شكل العناصر بدقة ونخلي التصميم يتكيف مع كل أحجام الشاشات.

Key Points

  1. Media Queries Basics:

    Media Queries بتخلي الموقع responsive يعني بيتأقلم مع حجم الشاشة (موبايل، تابلت، لابتوب...). بنستخدمها بالشكل ده:

    @media (max-width: 768px) { ... } الأكواد اللي جواها بتتطبق فقط لما الشاشة أقل من 768px.

    @media (min-width: 992px) { ... } الأكواد دي بتشتغل لما الشاشة أكبر من 992px.

  2. Min-width vs Max-width:

    • min-width بتستخدم لما نبدأ من الشاشات الصغيرة للأكبر (Mobile First).
    • max-width بتستخدم لما نبدأ من الشاشات الكبيرة وننزل للأصغر (Desktop First).

    💡 الأفضل إنك ترتب الـ media queries تصاعديًا حسب حجم الشاشة لتجنب التعارض.

  3. Common Breakpoints:

    • max-width: 576px للموبايلات الصغيرة.
    • max-width: 768px للأجهزة المتوسطة (تابلت).
    • max-width: 992px للشاشات الكبيرة.
    • max-width: 1200px للشاشات العريضة (ديسكتوب).

  4. Selectors in CSS:

    اتكلمنا عن أنواع الـ selectors اللي بنستخدمها لتحديد العناصر:

    • Element Selector p { } يختار كل الفقرات.
    • Class Selector .box { } يختار العناصر اللي فيها الكلاس ده.
    • ID Selector #main { } يختار عنصر واحد بالـ ID.
    • Universal Selector * { } يختار كل العناصر في الصفحة.
    • Group Selector h1, h2, h3 { } يطبق نفس التنسيق على أكتر من عنصر.
    • Descendant Selector div p { } ينسق الـ <p> اللي جواه <div>.
    • Child Selector div > p { } ينسق الأبناء المباشرين فقط.
    • Adjacent Sibling h2 + p { } ينسق العنصر اللي جاي بعد <h2> مباشرة.
    • General Sibling h2 ~ p { } ينسق كل العناصر اللي بعده في نفس المستوى.

  5. Pseudo Classes:

    هي حالات خاصة بنطبقها على العناصر حسب التفاعل:

    • :hover لما المستخدم يمرر الماوس على العنصر.
    • :focus لما العنصر زي <input> يكون نشط وجاهز للكتابة.
    • :active أثناء الضغط على العنصر.
    • :visited للروابط اللي المستخدم زارها قبل كده.
    • :first-child و:last-child لتنسيق أول وآخر عنصر داخل الأب.
    • :nth-child(2) لتحديد عنصر حسب ترتيبه.

  6. Input Selectors:

    في الجزء ده اتكلمنا عن المحددات الخاصة بعناصر الإدخال <input>، واللي بتسمح لنا ننسق الحالات المختلفة للفورم من غير جافاسكريبت.

    • input:disabled بيحدد الحقول المقفولة (غير قابلة للتعديل).
    • input:enabled بيحدد الحقول اللي شغالة وقابلة للكتابة.
    • input:required بيحدد الحقول الإلزامية اللي لازم المستخدم يملأها.
    • input:optional بيحدد الحقول الاختيارية.
    • input:focus بيُستخدم لتنسيق الحقل لما المستخدم يكون كاتب فيه أو محدده.

    الأمثلة دي بتوضح إزاي نقدر نعمل تنسيق تفاعلي للفورم زي تغيير اللون أو الحدود لما المستخدم يبدأ يكتب أو لما الحقل يكون مطلوب.

  7. Pseudo Elements:

    بتُستخدم لإضافة محتوى أو تنسيق وهمي من غير ما نضيف عناصر في الـ HTML:

    • ::before يضيف محتوى قبل العنصر.
    • ::after يضيف محتوى بعد العنصر.
    • ::first-letter لتنسيق أول حرف.
    • ::first-line لتنسيق أول سطر.
    • ::selection لتغيير شكل النص أثناء تحديده.

Notes

  • 💡 استخدم Mobile First approach باستخدام min-width لترتيب أفضل.
  • ⚙️ دايمًا حط الـ media queries في آخر ملف CSS علشان متغطيش التنسيقات السابقة.
  • 🎨 الـ pseudo elements بتساعدك تعمل تأثيرات احترافية من غير عناصر إضافية.
  • 🧠 الجمع بين الـ selectors و pseudo classes بيخليك تتحكم في أدق تفاصيل الصفحة.
