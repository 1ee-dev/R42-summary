Lecture 11: Specificity, Pseudo Classes & Attribute Selectors

Summary

المحاضرة دي كانت عن أولوية الأكواد في CSS، وازاي المتصفح بيختار أي تنسيق يطبقه لما يكون في أكتر من قاعدة بتأثر على نفس العنصر. كمان اتكلمنا عن Pseudo Classes و Pseudo Elements، واللي بيخلونا ننسق العنصر في حالات معينة أو نضيف عليه عناصر قبل وبعد من غير ما نلمس كود الـ HTML. وفي آخر المحاضرة، اتعرفنا على Attribute Selectors اللي بنستخدمها علشان ننسق العناصر على حسب الـ Attributes بتاعتها.

Key Points

  1. CSS Specificity:

    هي الطريقة اللي بيحسب بيها المتصفح الأولوية بين القواعد. لما يكون في أكتر من كود بيأثر على نفس العنصر، المتصفح بيختار الأقوى حسب الترتيب ده:

    • Inline Style أعلى أولوية (اللي مكتوب جوه التاج نفسه).
    • ID Selector بعده في القوة.
    • Class / Pseudo Class / Attribute Selector أقل من الـ ID.
    • Element Selector (زي <h2> أو <p>) الأضعف.

    مثال:

    <h2 id="title11" class="title10" style="background-color: brown;">Islam Mohamed</h2>

    اللون اللي هيظهر هو البني لأن inline style أقوى من أي حاجة تانية.

  2. Pseudo Classes:

    دي حالات معينة بتحصل للعنصر زي لما المستخدم يمرر الماوس أو يضغطه أو يكتف فيه.

    • :hover لما الماوس يقف على العنصر.
    • :active لما العنصر بيتضغط.
    • :focus لما العنصر زي input يكون جاهز للكتابة.

    مثال:

    .a1:hover { color: red; }

    يعني لما الماوس يقف على اللينك، لونه هيتحول لأحمر.

  3. Pseudo Elements:

    دي بتخلينا نضيف جزء قبل أو بعد العنصر من غير ما نعدل في كود الـ HTML.

    • ::before بيضيف محتوى قبل العنصر.
    • ::after بيضيف محتوى بعد العنصر.

    وغالبًا بنستخدمها مع الخاصية content: لتزيين العناصر أو إضافة رموز شكلية.

  4. Attribute Selectors:

    بنستخدمها لما نحب ننسق عناصر معينة على حسب الـ Attribute اللي فيها.

    مثال:

    [type="password"] { background-color: red; }
[type="text"] { background-color: greenyellow; }

    كده كل <input> نوعه password هيكون لونه أحمر، وكل <input> نوعه text هيكون أخضر فاتح.

  5. Display vs Visibility vs Opacity:

    الخصائص دي كلها بتتحكم في ظهور العنصر لكن كل واحدة بتشتغل بطريقة مختلفة:

    • display: none; العنصر بيختفي تمامًا ومش بياخد مساحة.
    • visibility: hidden; العنصر بيختفي لكن لسه واخد مكانه.
    • opacity: 0; العنصر بيبقى شفاف لكن لسه موجود في الصفحة.

  6. Specificity Calculation:

    المتصفح بيحسب الأولوية بالأرقام بالشكل ده:

    • Inline Style = 1000 نقطة.
    • ID = 100 نقطة.
    • Class / Pseudo / Attribute = 10 نقاط.
    • Element = 1 نقطة.

    وبيجمع الأرقام وبيطبّق القاعدة الأعلى. شفنا مثالين في الكود النتيجة فيهم كانت 476 مقابل 264 عشان نعرف طريقة الحساب.

Notes

  • 💡 استخدم أقل عدد ممكن من selectors علشان الكود يبقى بسيط وسهل التعديل.
  • ⚙️ الـ inline style دايمًا أقوى من أي كود تاني إلا لو استخدمت !important.
  • 🎯 استخدم pseudo-classes علشان تخلي الموقع تفاعلي من غير جافاسكريبت.
  • 🧠 فهم specificity هيساعدك تحل مشاكل تضارب التنسيقات بسرعة واحترافية.
